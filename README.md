# panel-wcschedule-guide-43c8

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存储，不针对任何特定网站或域名，仅作为通用页面集合进行维护。

## 目录说明

```
/
├── index.html          # 入口页面（可选）
├── pages/              # 主要页面存放目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录存放所有独立页面文件，文件名应反映页面用途。
- `assets/` 目录存放页面引用的静态资源，建议按类型分文件夹管理。
- 根目录下的 `index.html` 可作为导航页或默认入口，非必须。

## 页面归档说明

- 所有页面均为纯静态 HTML，不依赖后端服务。
- 页面之间相互独立，无跨页面依赖关系。
- 归档内容可能涉及不同主题或功能模块，请参考页面文件名或内部注释了解用途。
- 页面资源（CSS、JS、图片）建议使用相对路径引用，以确保在本地或 GitHub Pages 等静态托管环境中正常工作。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，同时更新相关资源文件至 `assets/` 对应子目录。
- 修改页面：直接编辑对应 HTML 文件，注意保持资源路径正确。
- 删除页面：移除对应文件，并清理不再使用的资源（如有）。
- 不建议在页面中嵌入外部链接或依赖第三方服务，以保持归档的独立性和稳定性。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件，或通过本地静态服务器预览。
3. 若部署至 GitHub Pages，请确保仓库设置中启用 Pages 功能，并选择正确的源分支。

## 许可

本项目仅用于归档目的，无特定许可证声明。如需引用或分发，请自行评估内容合规性。
